About this service

This accessibility statement applies to the Apply to register as a Companies House authorised agent service.

This service is run by Companies House. We want as many people as possible to be able to use this service. For example, that means you should be able to:

  • change colours, contrast levels and fonts using browser or device settings
  • zoom in up to 400% without the text spilling off the screen
  • navigate most of the website using a keyboard or speech recognition software
  • listen to most of the website using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

We've also made the service text as simple as possible to understand.

Find out more about accessibility support for Companies House users.

How accessible this service is

We know some parts of this service are not fully accessible:

  • the language setting of some pages cannot be determined by screen readers or other assistive technologies
  • some pages have titles that do not uniquely describe their topic or purpose
  • some pages require the user to re-enter data they have previously entered
  • some pages lack a correct heading hierarchy
  • some pages lack correctly labelled form fields
  • some forms do not feature the necessary fieldset component
  • one page prevents users from copying and pasting data
  • some pages contain links which do not clearly describe their purpose or destination
  • some pages feature expandable content which does not convey to assistive technologies whether it is in a collapsed or expanded state
  • one page visually presents information as a paired list, but the list does not have the semantic markup needed to convey the relationship to assistive technologies
  • the focus order of some pages does not follow a logical order
  • some images do not have descriptive alternative text
  • some error messages lack accurate or sufficient descriptions

Technical information about this service's accessibility

Companies House is committed to making its website accessible in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

This website is partially compliant with the Web Content Accessibility Guidelines version 2.2 AA standard, due to the non-compliances listed below.

Non-accessible content

The content listed below is non-accessible for the following reasons.

Non-compliance with the accessibility regulations

  1. The default human language of some pages cannot be programmatically determined. This fails WCAG 2.2 success criterion 3.1.1 Language of Page (Level A) Understanding Language of Page. We plan to fix this issue by April 2025.
  2. Some pages do not have titles that uniquely described their topic or purpose. This fails WCAG 2.2 success criterion 2.4.2 Page Titled (Level A). We plan to fix this issue by April 2025.
  3. On some pages, previously entered data has not been used to pre-populate form fields to avoid redundant re-entry. This fails WCAG 2.2 success criteria 3.3.7 Redundant Entry (Level A) and 4.1.2 Name, Role, Value (Level A). We plan to fix this issue by April 2025.
  4. On some pages, information, structure, and relationships conveyed through presentation cannot be programmatically determined or is not available in text. This fails WCAG 2.2 success criterion 1.3.1 Info and Relationships (Level A). We plan to fix this issue by April 2025.
  5. On some pages, unlabelled form fields are present. This fails WCAG 2.2 success criteria 1.3.1 Info and Relationships (Level A), 2.5.3 Label in Name (Level A), and 4.1.2 Name, Role, Value (Level A). We plan to fix this issue by April 2025.
  6. On some pages, related form controls are not grouped within a fieldset. This fails WCAG 2.2 success criterion 1.3.1 Info and Relationships (Level A). We plan to fix this issue by April 2025.
  7. An email confirmation prevents users from pasting data into the input. This fails WCAG 2.2 success criterion 3.3.7 Redundant Entry (Level A). We plan to fix this issue by April 2025.
  8. On some pages, text which visually labels a component does not correspond with the words associated with the component programmatically. This fails WCAG 2.2 success criterion 2.5.3 Label in Name (Level A). We plan to fix this issue by April 2025.
  9. Some links are not descriptive of their purpose or destination. This fails WCAG 2.2 success criterion 2.4.4 Link Purpose (In Context) (Level A). We plan to fix this issue by April 2025.
  10. On some pages, expandable content lacks the aria-expanded attribute to inform screen reader users of its state. This fails WCAG 2.2 success criterion 4.1.2 Name, Role, Value (Level A). We plan to fix this issue by April 2025.
  11. On one page, content visually presented or logically acting as a list does not use programmatic list markup to equivalently convey this to screen reader users. This fails WCAG 2.2 success criterion 1.3.1 Info and Relationships (Level A). We plan to fix this issue by April 2025.
  12. On some pages, focusable components do not receive focus in an order that preserves meaning and operability. This fails WCAG 2.2 success criterion 2.4.3 Focus Order (Level A). We plan to fix this issue by April 2025.
  13. On some pages, there are nested main landmarks on the page that are not uniquely identifiable. This fails WCAG 2.2 success criteria 1.3.1 Info and Relationships (Level A) and 4.1.2 Name, Role, Value (Level A). We plan to fix this issue by April 2025.
  14. Informative images are present that are not accurately conveyed to assistive technologies. This fails WCAG 2.2 success criterion 1.1.1 Non-text Content (Level A). We plan to fix this issue by April 2025.
  15. Some error messages provided are not sufficiently descriptive for users to determine what is required, as abbreviations that are not expanded or explained on their first are used. This fails WCAG 2.2 success criterion 3.3.3 Error Suggestion (Level AA). We plan to fix this issue by April 2025.
  16. On some pages, error messages provided do not provide appropriate suggestions for correction. This fails WCAG 2.2 success criterion 3.3.3 Error Suggestion (Level AA). We plan to fix this issue by April 2025.
  17. Some pages contain identical legends and labels which do not sufficiently describe their unique context. This fails WCAG 2.2 success criterion 2.4.6 Headings and Labels (Level AA). We plan to fix this issue by April 2025.

What we're doing to improve accessibility

To improve this service's accessibility, we:

  • work with real users who have accessibility issues to make sure that our services meet their needs
  • carry out internal tests for compliance with Web Content Accessibility Guidelines V2.2 level A and level AA
  • recruit users through our user panel and agencies to take part in usability sessions
  • build and test our services to meet government accessibility requirements

Read our blogs about usability to find out more about our user research work.

Preparation of this accessibility statement
